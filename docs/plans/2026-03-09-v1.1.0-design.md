# cortexmem v1.1.0 Design ‚Äî "Close the Gap"

**Date:** 2026-03-09
**Status:** Approved
**Parent:** `docs/plans/2026-03-09-roadmap-design.md`

## Overview

v1.1.0 closes the feature gap with Engram (agent-teams-lite's memory backend) by adding: HTTP API, `mem_save_prompt`, hard delete, Homebrew formula, and improved SKILL.md.

All entry points reuse `CortexMemServer.call_*()` ‚Äî zero logic duplication:

```
HTTP Request ‚Üí axum handler ‚Üí Arc<CortexMemServer>.call_*() ‚Üí SQLite
MCP Request  ‚Üí rmcp handler ‚Üí &CortexMemServer.call_*()     ‚Üí SQLite
CLI Command  ‚Üí cli handler  ‚Üí CortexMemServer.call_*()      ‚Üí SQLite
```

## Feature 1: HTTP API ‚Äî `cortexmem serve`

### Dependencies

```toml
axum = "0.8"
tower-http = { version = "0.6", features = ["cors"] }
```

### Subcommand

```
cortexmem serve [--port 7437] [--host 127.0.0.1]
```

### Architecture

`Arc<CortexMemServer>` shared via axum `State`. Same `call_*` methods used by CLI and MCP. No auth (localhost-only for now).

### New Files

- `src/http/mod.rs` ‚Äî axum router, app builder, server startup
- `src/http/routes.rs` ‚Äî endpoint handler functions

### Endpoints

| Method | Path | Maps to | Description |
|--------|------|---------|-------------|
| GET | `/health` | ‚Äî | `{"status":"ok","version":"..."}` |
| POST | `/observations` | `call_save` | Create observation |
| GET | `/observations/:id` | `call_get` | Full observation by ID |
| PATCH | `/observations/:id` | `call_update` | Partial update |
| DELETE | `/observations/:id` | `call_delete` | Soft or hard delete `?hard=true` |
| GET | `/search` | `call_search` | `?q=&type=&project=&scope=&limit=` |
| GET | `/context` | `call_context` | `?project=X&scope=` |
| GET | `/timeline` | `call_timeline` | `?observation_id=N&window=5` |
| POST | `/sessions` | `call_session_start` | `{project, directory}` |
| POST | `/sessions/:id/end` | `call_session_end` | `{summary?}` |
| GET | `/stats` | `call_stats` | `?project=X` |
| POST | `/compact` | `call_compact` | `?project=X` |
| GET | `/export` | export logic | `?project=X` JSON dump |
| POST | `/import` | import logic | JSON body import |
| POST | `/prompts` | `call_save_prompt` | Save user prompt |
| GET | `/prompts/recent` | `call_recent_prompts` | `?project=X&limit=N` |

### Concurrency

`Mutex<MemoryManager>` serializes writes. WAL mode allows concurrent reads. Sufficient for local dev tool. If contention becomes an issue in the future, we can move to `tokio::sync::RwLock`.

### Error Handling

All handlers return `Result<Json<T>, AppError>` where `AppError` implements `IntoResponse` with proper HTTP status codes:

```rust
enum AppError {
    NotFound(String),
    BadRequest(String),
    Internal(anyhow::Error),
}
```

### CLI Integration

Add `Serve` variant to `Commands` enum:

```rust
Serve {
    #[arg(short, long, default_value = "7437")]
    port: u16,
    #[arg(long, default_value = "127.0.0.1")]
    host: String,
},
```

## Feature 2: `mem_save_prompt` ‚Äî User Prompt Storage

### Schema Migration v2

```sql
CREATE TABLE IF NOT EXISTS user_prompts (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id  INTEGER REFERENCES sessions(id),
    content     TEXT NOT NULL,
    project     TEXT,
    created_at  TEXT DEFAULT (datetime('now'))
);

CREATE VIRTUAL TABLE IF NOT EXISTS prompts_fts USING fts5(
    content, project,
    content=user_prompts,
    content_rowid=id,
    tokenize='porter unicode61'
);
```

Migration is additive ‚Äî existing data untouched. `CURRENT_VERSION` bumps from 1 to 2.

### New MCP Tool

```
mem_save_prompt(content: String, session_id?: i64, project?: String)
```

### New DB Methods

- `insert_prompt(session_id, content, project) -> Result<i64>`
- `get_recent_prompts(project, limit) -> Result<Vec<Prompt>>`
- `search_prompts(query, project, limit) -> Result<Vec<Prompt>>`
- `sync_prompt_to_fts(id) -> Result<()>`

### New Struct

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Prompt {
    pub id: i64,
    pub session_id: Option<i64>,
    pub content: String,
    pub project: Option<String>,
    pub created_at: String,
}
```

### Integration

`mem_context` response updated to include recent prompts section:

```
Recent prompts:
- [2026-03-09 10:30] "Add authentication to the API"
- [2026-03-09 10:15] "Fix the login bug"
```

### New Files

- `src/db/prompts.rs` ‚Äî DB methods for prompts table

## Feature 3: Hard Delete

### Parameter Change

```rust
#[derive(Debug, Deserialize, JsonSchema)]
pub struct MemDeleteParams {
    pub id: i64,
    #[serde(default)]
    pub hard: Option<bool>,
}
```

### Behavior

- `hard: false/None` (default) ‚Üí soft delete: `UPDATE observations SET deleted_at = datetime('now') WHERE id = ?`
- `hard: true` ‚Üí permanent delete:
  1. Remove from `observations_fts` (using FTS5 delete command)
  2. Remove from `vec_observations`
  3. `DELETE FROM observations WHERE id = ?`

### New DB Method

```rust
pub fn hard_delete_observation(&self, id: i64) -> Result<()>
```

### CLI

Add `Delete` variant to `Commands`:

```rust
Delete {
    id: i64,
    #[arg(long)]
    hard: bool,
},
```

## Feature 5: SKILL.md Improvements

### Progressive Disclosure Enforcement

Add explicit rule:
> "NEVER use `mem_search` results directly as complete artifacts. Always follow up with `mem_get` to retrieve full untruncated content."

### Naming Convention Guide

Add recommended `topic_key` patterns:

```
architecture/{decision}   ‚Äî e.g., architecture/auth-model
bug/{issue-slug}          ‚Äî e.g., bug/login-redirect-loop
pattern/{name}            ‚Äî e.g., pattern/error-handling
decision/{choice}         ‚Äî e.g., decision/use-axum-over-actix
config/{system}           ‚Äî e.g., config/database-connection
discovery/{finding}       ‚Äî e.g., discovery/sqlite-wal-locks
learning/{topic}          ‚Äî e.g., learning/rust-lifetimes
milestone/{release}       ‚Äî e.g., milestone/v1-1-0-release
```

### Workflow State Persistence

Add pattern for multi-agent orchestrators:

```
Save DAG/workflow state as a memory artifact:
  topic_key: workflow/{change-name}/state
  type: architecture
  content: serialized state (phase, completed steps, artifact refs)
```

This enables recovery after context compaction ‚Äî agents reconstruct state by reading the topic key instead of re-deriving it.
