# cortexmem Roadmap Design — v1.3.0 through v1.5.0

**Date:** 2026-03-09
**Status:** Approved

## Overview

Three releases evolving cortexmem from a dev tool into a widely distributed, intelligent memory platform with ecosystem integrations.

| Release | Theme | Features |
|---------|-------|----------|
| **v1.3.0** | Distribution & Adoption | Homebrew, npm wrapper, shell installer, Docker image, `cortexmem doctor`, setup wizard improvements |
| **v1.4.0** | Intelligence Upgrade | Multi-model embeddings, auto-tagging, relevance learning |
| **v1.5.0** | Ecosystem & Extensibility | REST API auth, TS + Python SDKs, webhooks |

## v1.3.0 — Distribution & Adoption

### Feature 1: Homebrew Formula & Tap

**Tap repo:** https://github.com/pablocalofatti/homebrew-tap (already created)

**Install:** `brew install pablocalofatti/tap/cortexmem`

**Formula:** `Formula/cortexmem.rb` downloads pre-built binaries from GitHub Releases per platform (darwin-arm64, darwin-x64, linux-arm64, linux-x64).

**SHA256 auto-update:** GitHub Action in the tap repo triggered by cortexmem release webhook. Fetches new tarballs, computes hashes, updates formula, commits.

**Changes to cortexmem repo:**
- Update `release.yml` to produce `.tar.gz` assets per platform
- Add `scripts/update-homebrew.sh` for manual formula updates as fallback

**Tap repo structure:**
```
homebrew-tap/
  Formula/
    cortexmem.rb
  .github/workflows/
    update-formula.yml
```

### Feature 2: npm/npx Wrapper

**Install:** `npm install -g cortexmem` or `npx cortexmem setup`

**Location:** `npm/` directory in cortexmem repo (monorepo style, keeps versions in sync).

**Package structure:**
```
npm/
  package.json
  bin/
    cortexmem.js          # shim that spawns the real binary via execFileSync (safe, no shell)
  scripts/
    install.js            # postinstall: detect platform, download binary
```

**How it works:**
1. `postinstall` hook runs `scripts/install.js`
2. Detects OS + arch via `process.platform` and `process.arch`
3. Downloads matching binary from GitHub Releases for the package version
4. Stores binary in package directory
5. `cortexmem.js` is a Node shim using `execFileSync` (not exec — no shell injection risk)

**Platform mapping:**
```js
const PLATFORMS = {
  'darwin-arm64': 'cortexmem-darwin-arm64.tar.gz',
  'darwin-x64':   'cortexmem-darwin-x64.tar.gz',
  'linux-arm64':  'cortexmem-linux-arm64.tar.gz',
  'linux-x64':    'cortexmem-linux-x64.tar.gz',
};
```

**Publishing:** npm publish added to release workflow, triggered after binaries are uploaded. Version synced with Cargo.toml.

### Feature 3: Shell Installer

**Install:** `curl -fsSL https://raw.githubusercontent.com/pablocalofatti/cortexmem/main/scripts/install.sh | sh`

**File:** `scripts/install.sh` in cortexmem repo.

**Script flow:**
1. Detect OS (`uname -s`) and arch (`uname -m`)
2. Fetch latest release tag from GitHub API
3. Download matching binary tarball from GitHub Releases
4. Extract to `~/.cortexmem/bin/cortexmem`
5. Add to PATH (append to `~/.bashrc` / `~/.zshrc` if not already present)
6. Run `cortexmem --version` to verify
7. Print next steps

**Edge cases:** Already installed (offer upgrade), no curl (wget fallback), unsupported platform (clear error).

### Feature 4: Docker Image

**Scope:** Cloud server only.

**Image:** `ghcr.io/pablocalofatti/cortexmem:latest` + version tags

**Dockerfile (multi-stage):**
```dockerfile
FROM rust:1.83-slim AS builder
WORKDIR /app
COPY . .
RUN cargo build --release --features cloud

FROM debian:bookworm-slim
RUN apt-get update && apt-get install -y ca-certificates && rm -rf /var/lib/apt/lists/*
COPY --from=builder /app/target/release/cortexmem /usr/local/bin/
EXPOSE 8080
ENTRYPOINT ["cortexmem", "cloud", "serve"]
```

**Publishing:** Build and push image on tag push in release workflow.

**Updated docker-compose.yml references published image instead of local build.**

### Feature 5: `cortexmem doctor`

**Command:** `cortexmem doctor [--fix]`

**Checks performed:**

| Check | Pass | Fail |
|-------|------|------|
| Binary version | Version string | — (always passes) |
| Database accessible | Path + size | Can't open / doesn't exist |
| Schema version | Current version | Outdated / corrupt |
| Embedding model | Downloaded, correct dims | Not downloaded |
| FTS5 index | Row count | Missing / empty |
| Vector index | Row count | Missing / empty |
| MCP config detected | Which agent(s) found | No config found |
| MCP config valid | Points to correct binary | Wrong path / missing |
| Cloud sync configured | Server URL + last sync | Not configured (warn) |
| Git sync configured | Repo path + status | Not configured (warn) |

**Output format:**
```
cortexmem doctor v1.3.0

  [OK]   Database: ~/Library/Application Support/cortexmem/cortexmem.db (2.4 MB)
  [OK]   Schema: v3
  [OK]   Embedding model: AllMiniLML6V2 (384-dim)
  [OK]   FTS5 index: 142 observations indexed
  [OK]   Vector index: 138 embeddings stored
  [OK]   MCP config: Claude Code (cortexmem mcp)
  [WARN] Cloud sync: not configured
  [WARN] Git sync: not configured

8 checks passed, 0 failed, 2 warnings
```

**`--fix` flag:** Download model, reindex FTS, launch setup wizard as needed.

**Implementation:** New file `src/cli/doctor.rs`, adds `Doctor` variant to `Commands`.

### Feature 6: Setup Wizard Improvements

**Improvements over current `src/cli/setup.rs`:**

**1. Auto-detection:** Scan known config paths, show detected vs not-detected agents.

**2. Validate before writing:** Check existing config, verify binary path, show JSON diff.

**3. Post-setup verification:** Spawn `cortexmem mcp`, send `initialize` JSON-RPC, verify response.

**4. New agents:**

| Agent | Config Location |
|-------|----------------|
| Claude Code | `~/.claude.json` → `mcpServers` |
| OpenCode | `~/.config/opencode/config.json` → `mcpServers` |
| Cursor | `~/.cursor/mcp.json` |
| Windsurf | `~/.codeium/windsurf/mcp_config.json` |
| VS Code | `.vscode/mcp.json` (workspace-level) |
| Gemini CLI | `~/.gemini/settings.json` |
| Zed | `~/.config/zed/settings.json` |
| Cline | `.vscode/cline_mcp_settings.json` |

**Implementation:** Update existing `src/cli/setup.rs`.

### Cross-cutting: Release Workflow Updates

Update `release.yml` to:
1. Build binaries → tar.gz per platform
2. Upload to GitHub Releases
3. Publish npm package (`cd npm && npm publish`)
4. Build and push Docker image (`ghcr.io`)
5. Trigger Homebrew tap update (repository dispatch)

## v1.4.0 — Intelligence Upgrade

### Feature 7: Multi-Model Embedding Support

**Configurable model via `~/.config/cortexmem/config.toml`:**

| Model | Dimensions | Size | Quality | Speed |
|-------|-----------|------|---------|-------|
| AllMiniLML6V2 | 384 | 23 MB | Good | Fast |
| BGE-Small-EN-v1.5 | 384 | 33 MB | Better | Fast |
| AllMiniLML12V2 | 384 | 46 MB | Better | Medium |
| NomicEmbedTextV1 | 768 | 274 MB | Best | Slow |

**Config:**
```toml
[embedding]
model = "AllMiniLML6V2"
```

**Migration on model change:** `cortexmem reindex` drops all vectors and re-embeds with new model. `doctor` detects model mismatch.

**Schema:** `embedding_model` in `meta` table tracks current model.

**Implementation:** Update `src/embed/model.rs`, new `src/config.rs`, new `cortexmem reindex` command.

### Feature 8: Auto-Tagging

**TF-IDF keyword extraction when `concepts`/`facts` not provided by agent.**

**Algorithm:**
1. Tokenize, remove stop words, simple suffix stemming
2. Score by TF × IDF (IDF from FTS5 term frequencies)
3. Top 5-8 terms → `concepts`
4. Extract declarative sentences → candidate `facts` (top 3)

**Runs on:** `mem_save` (if concepts empty), `mem_update` (if content changed), `cortexmem auto-tag --all` (batch).

**Implementation:** New `src/memory/autotag.rs`, static stop words `HashSet`.

### Feature 9: Relevance Learning

**Track search→access signal, boost future rankings.**

**New table (migration v4):**
```sql
CREATE TABLE IF NOT EXISTS search_feedback (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    query_text      TEXT NOT NULL,
    observation_id  INTEGER REFERENCES observations(id),
    session_id      INTEGER REFERENCES sessions(id),
    clicked         BOOLEAN DEFAULT TRUE,
    created_at      TEXT DEFAULT (datetime('now'))
);
```

**Ranking boost:** After RRF fusion, multiply score by `1.0 + 0.1 * feedback_count` (capped at 2.0) for observations with positive feedback on similar queries.

**Implementation:** New `src/db/feedback.rs`, update `src/search/mod.rs` and `mem_get` handler.

## v1.5.0 — Ecosystem & Extensibility

### Feature 10: REST API Auth

**Optional API key auth for `cortexmem serve`.**

```toml
[http]
auth = true
```

On first `cortexmem serve --auth`, generate and print a local API key. Store hashed (SHA-256). All endpoints except `/health` require `Authorization: Bearer <key>`.

**Implementation:** Auth middleware in `src/http/routes.rs`, `cortexmem api-key [regenerate]` command.

### Feature 11: TypeScript + Python SDKs

**Thin HTTP clients wrapping the REST API.**

**Location:** `sdks/typescript/` and `sdks/python/` in cortexmem repo.

**Features:** Type-safe models, auto-retry with backoff, zero deps beyond HTTP client.

**Publishing:** npm (`@cortexmem/sdk`) and PyPI (`cortexmem`) via release workflow.

### Feature 12: Webhooks

**Config-driven HTTP POST notifications on events.**

```toml
[[webhooks]]
url = "https://hooks.slack.com/services/..."
events = ["observation.saved", "session.ended"]
```

**Events:** `observation.saved`, `observation.updated`, `observation.deleted`, `session.started`, `session.ended`, `compaction.completed`.

**Delivery:** Fire-and-forget, 3 retries with backoff.

**Implementation:** New `src/hooks/webhooks.rs`, config module reads `cortexmem.toml`.
